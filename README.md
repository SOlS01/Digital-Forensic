# Association of chief police officers(ACPO) >> it is a UK professional body established in 1964, it provide guidance and support for polices services in a wide range. 

# 4 principle of ACPO : 
responsibility is on the investigator shoulder, the person must make sure all the rules followed.
No action should change data, evidence must not be altered when accessed
If access needed it must be justified and only trained people should access to it and must explain what is done.
Must keep track of all the actions taken so everyone can review it.

4 principle of information management :
proportionate >> information should be collected, processed and used only if it is necessary for policing purposes.
Retention >> information should be retained as long as it is necessary unless it should be deleted.
Accessibility >> information should be easy accessible for specific policing purpose and a security measure should be placed to protect the confidentiality of the information.
Transparency >> the collection, processing and storing the information should be in a way that is transparent and open to public and the individual must have access to their own information.

RIPA(regulation of investigatory power act) >> it govern the investigators power like surveillance, interception, using information by public authorities. It make sure that the power used is proportional and necessary and it was in a way that respects human rights.

# 3 main stages of the digital forensic investigatory process >>

incident response: this stage is about collecting as much as information possible and preserving digital evidence, we focus on the scope of the incident and minimise the incident impact. It involves prelim assessment, collect information about devices and systems, taking necessary steps to not damage or loss the evidence.
Managing digital evidence: at this stage we focus on preserving and analysing the evidences, gathering as much information as possible and find the pattern that may indicate the cause of the incident, this stage involve making a forensically sound copy of the evidence and analyse the system and network to find the cause of the incident
Preparing forensic documents: at this stage we focus on method used, the result of the investigation and providing accurate numbers and records about the investigation in a clear and concise way.

# Digital forensic investigatory techniques >>

hash : it is a fixed length digital signature which cover the data into a coded unreadable format, with this way we can verify that the data hasn’t been altered in any way and making sure about integrity and reliability.
Chain of custody : documenting step by step of collecting data from the very first to the last position. With the chain of custody it will be clear who/when has access to the evidence and what has been done to it and where it is stored.
Contemporaneous notes : it is providing accurate records of every steps and what happened and it help the investigator to remember what they have done so far, this notes shows how the data handled, it can be used as a proof in the court because it has got time and details.

# Steganography >> a technique and method to hide important things behind an image and files, so it is really important that the investigator be aware of file/image compartment and be able to use tools such as Autopsy.

# Acquisition >> there is 2 different acquisition:

- live : capturing data from a running system, pros: we can have access to the encrypted files and volatile data which will be gone after the system is turned off. cons: it is hard to collect data from multiple sources , the person might damage the evidence or face data loss
dead : capturing data from a turned off system, pros: the potential damage is eliminated cons: the person can’t have access to the volatile data and encrypted data and locked system

Source of digital evidence in digital forensics >> everything on a crime scene is an evidence.
